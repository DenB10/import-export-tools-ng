ImportExportTools NG 概述和菜单

ImportExportTools NG 是基于 Paolo “Kaosmos” 原创作品的后续扩展。该扩展提供了多种导入和导出功能，支持多种不同的文件格式，包括： 注意：并非所有功能都支持某些格式。
- mbox
- EML
- 邮件目录
- HTML
- 纯文本
- CSV（逗号分隔值）
- PDF

虽然大部分功能都围绕消息展开，但也支持导入和导出 Thunderbird 配置文件。此外，导出还可以包括 HTML 或 CSV 索引文件。 Thunderbird 配置文件的备份是另一个功能。

访问 IETng 功能是通过扩展安装的几个不同菜单完成的。这些功能是上下文相关的。添加了以下菜单：
- The Folder Pane Context（文件夹操作）
- 消息列表上下文（选定的消息）
- 工具菜单（配置文件和备份操作）

“工具菜单”下可访问的选项不在“汉堡包”菜单中，它们将在未来的版本中。这些功能在新的工具栏按钮中也可用。每个菜单和按钮还有一个帮助和选项菜单项，用于访问选项对话框。

自定义日期格式

自定义日期选项允许完全控制用于索引中两个文件的日期格式。默认格式“日期”和新的自定义格式“日期（自定义）”都可以在文件名下拉列表中找到。自定义日期也可以与扩展自定义文件名中使用的 ${date_custom} 标记一起使用。原始（默认）日期格式为：
YYYYMMDDHHmm : 202011060808

自定义日期格式使用与strftime()类 Unix 功能相匹配的 % 序列。它基于strftime()的 JavaScript 端口，用于格式化日期和时间的函数。
TH Doan 的 strftime() 端口 (https://thdoan.github.io/strftime/)

转换顺序说明

顺序 描述
%a 星期几的缩写名称。
%A 星期几的全称。
%b 缩写的月份名称。
%B 完整的月份名称。
%c 当前语言环境的首选日期和时间 (UTC) 表示。
%C 世纪数（年/100）作为 2 位整数。
%d 以十进制数表示的月份中的第几天（范围从 01 到 31）。
%e 以十进制数表示的月份中的第几天（范围从 1 到 31）。
%F ISO 8601 日期格式（相当于%Y-%m-%d ）。
%G ISO 8601 基于周的年份，世纪作为十进制数。 4 位年份对应于 ISO 周数（请参阅%V ）。这与%Y具有相同的格式和值，除了如果 ISO 周数属于上一年或下一年，则使用那一年。
%g 类似于%G ，但没有世纪，即具有两位数的年份 (00-99)。
%H 使用 24 小时制（范围为 00 到 23）的十进制数形式的小时。另见%k 。
%I 使用 12 小时制（范围为 01 到 12）的十进制数形式的小时。另见%l 。
%j 以十进制数表示的一年中的第几天（范围从 001 到 366）。
%k 使用 24 小时制（范围为 0 到 23）的十进制数形式的小时。另见%H 。
%l 使用 12 小时制（范围为 1 到 12）的十进制数形式的小时。另见%I 。
%m 十进制数形式的月份（范围为 01 到 12）。
%n 十进制数形式的月份（范围为 1 到 12）。
%M 十进制数形式的分钟（范围为 00 到 59）。
%p 根据给定的时间值，可以是“AM”或“PM”。中午被视为“PM”，午夜被视为“AM”。
%P 像%p但小写（“am”或“pm”）。
%s 自纪元 1970-01-01 00:00:00 +0000 (UTC) 以来的秒数。
%S 秒为十进制数（范围为 00 到 59）。
%u 小数形式的星期几（范围 1 到 7），星期一为 1。另请参见%w 。
%V ISO 8601 十进制数形式的当年的 ISO 8601 周数（范围 01 到 53），其中第 1 周是新年中至少有 4 天的第一周（即第一个星期四）。
%w 小数形式的星期几（范围 0 到 6），星期日为 0。另请参见%u 。
%x 不带时间的当前语言环境的首选日期表示。
%X 不带日期的当前语言环境的首选时间表示。
%y 不带世纪的十进制数字形式的年份（范围从 00 到 99）。
%Y 以十进制数字表示的年份，包括世纪。
%z +hhmm或-hhmm数字时区（即与 UTC 的小时和分钟偏移量）。
%Z 时区名称或缩写。

兼容性说明：

  • %c - 格式化字符串略有不同
  • %D - 未实现（使用%m/%d/%y或%d/%m/%y ）
  • %e - 不添加空间
  • %E - 未实施
  • %h - 未实现（使用%b ）
  • %k - 未添加空间
  • %n - 类似%m ，但没有前导零（使用\n换行）
  • %O - 未实施
  • %r - 未实现（使用%I:%M:%S %p ）
    • ${subject} 邮件主题
  • %R - 未实现（使用%H:%M ）
  • %t - 未实现（使用\t ）
  • %T - 未实现（使用%H:%M:%S ）
  • %U - 未实施
  • %W - 未实现
  • %+ - 未实施
  • %% - 未实现（使用% ）

扩展文件名格式使用标记。可以使用标记以及任何文本或有效的文件名字符来构造确切的文件名。无效字符是： | * : < > ? \ / "英语 (en-US) 标记在任何语言环境中都可用。特定于语言环境的标记也可用并列在英语表下方。

新的和实验性的偏好（选项）

在选项对话框中有几个新功能和选项尚不可配置。一旦这些功能稳定下来，它们就会被添加到 UI 中。
为了使用它们，必须在配置编辑器中进行设置。要访问编辑器，请使用“工具/选项”菜单，然后选择“高级”部分，底部会有一个“配置编辑器”按钮。如果您在搜索栏中键入“importexporttoolsng”，您将获得所有扩展首选项。然后，您可以根据需要编辑下面的各种首选项。

importexporttoolsng.export.index_date_custom_format

此首选项遵循用于索引文件的相同完整格式。默认情况下它是一个空字符串，如果带有格式化的日期/时间字符串，那么这将更改索引日期格式。

importexporttoolsng.export.attachments.filename_extended_format

importexporttoolsng.export.embedded_attachments.filename_extended_format

这两个首选项都使用与导出的消息文件名使用的相同类型的扩展文件命名格式。第一个用于外部附件，第二个用于嵌入式（内联）附件。

importexporttoolsng.experimental.use_delivery_date

通常，Thunderbird 中邮件的“日期”是发件人邮件客户端设置的日期。将此选项设置为“true”会利用收件人邮件客户端的接收日期。该日期将用于文件名以及索引文件中的日期。

基本的全局热键（快捷方式）功能

importexporttoolsng.experimental.hot_keys

这个新首选项允许用户设置最多 10 个键盘快捷键。这些快捷方式可以执行大多数 ImportExportTools NG 功能以及其他 Thunderbird 功能。首选项字符串被格式化为 JSON 格式的对象。格式和键值如下：

示例热键条目（JSON 对象数组）

id - 键索引 (1-10)
key - 键（单个字符）
keycode - 'VK_*' 定义键码
可以包括密钥或密钥代码，但不能同时包括两者。
modifiers - 键修饰符，空格分隔列表（'control', 'shift', 'alt', 'accel'）
oncommand - 直接命令字符串
contexts - 键处于活动状态的窗口上下文
'all' - Messenger、撰写、搜索下的所有窗口
'messenger' - 主 Messenger 窗口及其子窗口
'compose' - 消息撰写窗口
'search' - 搜索对话框
