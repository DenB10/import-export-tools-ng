ImportExportTools NG 개요 및 메뉴

ImportExportTools NG는 Paolo "Kaosmos"의 원본 작업을 기반으로 하는 후속 확장입니다. 확장 기능은 다음과 같은 여러 파일 형식을 지원하는 여러 가져오기 및 내보내기 기능을 제공합니다. 참고: 일부 형식은 모든 기능에서 지원되지 않습니다.
- 엠박스
- EML
- 메일디렉토리
- HTML
- 일반 텍스트
- CSV(쉼표로 구분된 값)
- PDF

대부분의 기능은 메시지를 중심으로 이루어지지만 Thunderbird 프로필 가져오기 및 내보내기도 지원합니다. 또한 내보내기에는 HTML 또는 CSV 인덱스 파일도 포함될 수 있습니다. Thunderbird 프로필 백업은 또 다른 기능입니다.

IETng 기능에 액세스하는 것은 확장 기능에 의해 설치된 여러 가지 메뉴를 통해 수행됩니다. 기능은 상황에 따라 다릅니다. 다음에 대한 메뉴가 추가되었습니다.
- 폴더 창 컨텍스트(폴더 작업)
- 메시지 목록 컨텍스트(선택된 메시지)
- 도구 메뉴(폴더 및 프로필 작업)

확장 옵션에 대한 "도구/애드온 옵션" 아래에 추가된 항목 아래에 액세스할 수 있는 많은 옵션이 있습니다. 각 메뉴에는 옵션 대화 상자에 액세스할 수 있는 옵션 메뉴 항목도 있습니다.
새 옵션은 여기에 문서화되어 있으며 전체 세트는 나중에 문서화됩니다.

맞춤 날짜 형식

사용자 지정 날짜 옵션을 사용하면 인덱스의 두 파일에 사용되는 날짜 형식을 완전히 제어할 수 있습니다. 파일 이름 드롭다운에서 기본 형식 "날짜"와 새로운 사용자 지정 형식 "날짜(사용자 정의)"를 모두 사용할 수 있습니다. 사용자 정의 날짜는 확장 사용자 정의 파일 이름에 사용되는 ${date_custom} 토큰과 함께 사용할 수도 있습니다. 원래(기본) 날짜 형식은 다음과 같습니다.
YYYYMMDDHHmm : 202011060808

사용자 지정 날짜 형식은 기능과 같은 유닉스 strftime() 과 일치하는 % 시퀀스를 사용합니다. 날짜 및 시간 형식을 지정하는 함수인 strftime() 의 JavaScript 포트를 기반으로 합니다.
TH Doan(https://thdoan.github.io/strftime/)의 strftime() 포트

전환 순서 설명

순서 설명
%a 요일의 약식 이름입니다.
%A 요일의 전체 이름입니다.
%b 축약된 월 이름입니다.
%B 전체 월 이름입니다.
%c 현재 로케일에 대한 기본 날짜 및 시간(UTC) 표현입니다.
%C 2자리 정수로 된 세기(연도/100).
%d 10진수 형식의 날짜(범위 01~31).
%e 10진수 형식의 날짜(범위 1~31).
%F ISO 8601 날짜 형식( %Y-%m-%d 와 동일).
%G 십진수로 세기를 사용하는 ISO 8601 주 기반 연도. 4자리 연도는 ISO 주 번호에 해당합니다( %V 참조). 이는 ISO 주 번호가 이전 연도 또는 다음 연도에 속하는 경우 해당 연도가 대신 사용된다는 점을 제외하면 %Y 와 형식 및 값이 동일합니다.
%g %G 와 비슷하지만 세기가 없습니다. 즉, 2자리 연도(00-99)가 있습니다.
%H 24시간제를 사용하는 10진수로 나타낸 시간(범위 00~23). %k 도 참조하십시오.
%I 12시간제를 사용하는 10진수로 나타낸 시간(범위 01~12). %l 도 참조하십시오.
%j 10진수 형식의 날짜(범위 001~366).
%k 24시간제를 사용하는 10진수로 나타낸 시간(0~23 범위). %H 도 참조하십시오.
%l 12시간제를 사용하는 10진수 형식의 시간(범위 1~12). %I 도 참조하십시오.
%m 월을 10진수로 표시합니다(범위 01~12).
%n 월을 10진수로 표시합니다(범위 1~12).
%M 10진수 형식의 분(범위 00~59).
%p 주어진 시간 값에 따라 "AM" 또는 "PM"입니다. 정오는 "PM"으로, 자정은 "AM"으로 처리됩니다.
%P %p 와 같지만 소문자("am" 또는 "pm")입니다.
%s Epoch, 1970-01-01 00:00:00 +0000(UTC) 이후의 초 수입니다.
%S 초는 10진수로 표시됩니다(범위 00~59).
%u 요일은 십진법(범위 1~7), 월요일은 1입니다 %w 도 참조하십시오.
%V 현재 연도의 ISO 8601 주 번호를 10진수(범위 01~53)로 표시합니다. 여기서 주 1은 새해에 최소 4일이 있는 첫 번째 주(즉, 첫 번째 목요일)입니다.
%w 십진수로 표시되는 요일(범위 0~6), 일요일은 0입니다. %u 도 참조하십시오.
%x 시간을 제외한 현재 로케일에 대한 기본 날짜 표현입니다.
%X 날짜가 없는 현재 로캘의 기본 시간 표현입니다.
%y 세기가 없는 10진수 연도(범위 00~99).
%Y 세기를 포함하는 십진수로 연도.
%z +hhmm 또는 -hhmm 숫자 시간대(즉, UTC로부터의 시간 및 분 오프셋).
%Z 시간대 이름 또는 약어입니다.

호환성 정보:

  • %c - 형식이 지정된 문자열이 약간 다릅니다.
  • %D - 구현되지 않음( %m/%d/%y 또는 %d/%m/%y 사용)
  • %e - 공백이 추가되지 않음
  • %E - 구현되지 않음
  • %h - 구현되지 않음( %b 사용)
  • %k - 공백이 추가되지 않음
  • %n - %m 과 비슷하지만 앞에 0이 없습니다(개행에 \n 사용).
  • %O - 구현되지 않음
  • %r - 구현되지 않음( %I:%M:%S %p 사용)
    • ${subject} 메시지의 제목
  • %R - 구현되지 않음( %H:%M 사용)
  • %t - 구현되지 않음( \t 사용)
  • %T - 구현되지 않음( %H:%M:%S 사용)
  • %U - 구현되지 않음
  • %W - 구현되지 않음
  • %+ - 구현되지 않음

확장된 파일 이름 형식:


확장된 파일 이름 형식은 토큰을 사용합니다. 정확한 파일 이름은 토큰과 텍스트 또는 유효한 파일 이름 문자를 사용하여 구성할 수 있습니다. 잘못된 문자는 다음과 같습니다. | * : < > ? \ / " 영어(en-US) 토큰은 모든 로캘에서 사용할 수 있습니다. 로캘별 토큰도 사용할 수 있으며 영어 표 아래에 나열되어 있습니다.

예:

${subject}-${date}-${index} - 기본 형식
${prefix}_${date_custom}_${sender} - 기본값 대신 사용자 지정 날짜 형식 사용
${subject}-GmailMsg-${date}-${index} - 맞춤 텍스트 포함


토큰 설명
${subject} 메시지의 제목입니다. 길이는 제목 길이 옵션에 따라 제한될 수 있습니다.
${sender} 메시지 발신자의 이름입니다.
${recipient} 메시지 수신자의 이름입니다.
${sender_email} 메시지 보낸 사람의 전자 메일 주소입니다.
${recipient_email} 메시지 수신자의 이메일 주소입니다.
${smart_name} 메시지가 보낸 편지함 폴더에 있으면 받는 사람, 그렇지 않으면 보낸 사람 이름
${index} 메시지 인덱스 번호입니다.
${prefix} 접두사 텍스트 옵션 값입니다.
${suffix} 접미사 텍스트 옵션 값입니다.
${date} 표준 날짜 형식입니다. 기본 형식은 YYYYMMDDHHmm입니다.
${date_custom} 사용자 지정 날짜 형식입니다. 사용자 정의 날짜 형식 섹션을 참조하십시오.

토큰(en-US) 현지화( ) 토큰
${subject}
${sender}
${recipient}
${sender_email}
${recipient_email}
${smart_name}
${index}
${prefix}
${suffix}
${date}
${date_custom}

새롭고 실험적인 기본 설정(옵션)

옵션 대화 상자 내에서 아직 구성할 수 없는 몇 가지 새로운 기능과 옵션이 있습니다. 이러한 기능이 안정화되면 UI에 추가됩니다.
사용하려면 구성 편집기 내에서 설정해야 합니다. 편집기에 액세스하려면 도구/옵션 메뉴를 사용한 다음 고급 섹션을 선택하면 하단에 '구성 편집기' 버튼이 있습니다. 검색창에 'importexporttoolsng'를 입력하면 모든 확장 기본 설정이 표시됩니다. 그런 다음 필요에 따라 아래의 다양한 기본 설정을 편집할 수 있습니다.

importexporttoolsng.export.index_date_custom_format

이 기본 설정은 색인 파일에 사용되는 것과 동일한 전체 형식을 따릅니다. 기본적으로 빈 문자열이며 형식이 지정된 날짜/시간 문자열이 있는 경우 인덱스 날짜 형식이 변경됩니다.

importexporttoolsng.export.attachments.filename_extended_format

importexporttoolsng.export.embedded_attachments.filename_extended_format

이러한 기본 설정은 모두 내보낸 메시지 파일 이름이 사용하는 것과 동일한 유형의 확장된 파일 이름 지정 형식을 사용합니다. 첫 번째는 외부 첨부 파일용이고 두 번째는 내장(인라인) 첨부 파일용입니다.

importexporttoolsng.experimental.use_delivery_date

일반적으로 Thunderbird에서 메시지의 '날짜'는 보낸 사람의 메일 클라이언트에서 설정한 날짜입니다. 이 옵션을 'true'로 설정하면 수신자의 메일 클라이언트에서 받은 날짜를 활용합니다. 이 날짜는 파일 이름과 인덱스 파일의 날짜 모두에 사용됩니다.

기본, 글로벌 핫키(바로가기) 기능

importexporttoolsng.experimental.hot_keys

이 새로운 기본 설정을 통해 사용자는 최대 10개의 키보드 단축키를 설정할 수 있습니다. 단축키는 대부분의 ImportExportTools NG 기능과 기타 Thunderbird 기능을 수행할 수 있습니다. 기본 설정 문자열은 JSON 형식 개체로 형식이 지정됩니다. 형식과 키는 다음과 같습니다.

단축키 항목 예(JSON 객체 배열)

id - 키 인덱스(1-10)
key - 키(단일 문자)
keycode - 'VK_*'는 키코드를 정의합니다.
키 또는 키코드를 포함할 수 있지만 둘 다 포함할 수는 없습니다.
modifiers - 키 수정자, 공백으로 구분된 목록('control', 'shift', 'alt', 'accel')
oncommand - 직접 명령 문자열
contexts - 키가 활성 상태인 창 컨텍스트
'all' - 메신저, 작성, 검색 아래의 모든 창
'messenger' - 기본 메신저 창과 그 하위 창
'compose' - 메시지 작성 창
'search' - 검색 대화상자
